# Firebase Schema - FloodGuard

This document outlines the Firestore collection structure used by the FloodGuard application.

## Collections

### `locations`
Stores live coordinates shared by users for the Rescue Dashboard.

- **Document ID**: Unique `client_id` (generated by the backend or frontend).
- **Fields**:
  - `lat` (number): Latitude of the user.
  - `lng` (number): Longitude of the user.
  - `timestamp` (timestamp): Last updated time.
  - `name` (string, optional): User's name if provided.

### `alerts`
(Optional) Stores history of triggered alerts.

- **Document ID**: Auto-generated.
- **Fields**:
  - `message` (string): The alert message.
  - `timestamp` (timestamp): When the alert was triggered.
  - `type` (string): Type of alert (e.g., 'flood', 'test').

## Configuration

### Frontend
The frontend connects to Firebase using the web configuration found in `assets/js/config.js`.

### Backend
The backend uses the Firebase Admin SDK initialized with `serviceAccountKey.json` located at the root of the `backend/` directory.
